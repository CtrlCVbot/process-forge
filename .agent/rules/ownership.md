## 1. 개요

이 규칙은 Process Forge 프로젝트의 **문서 소유권 원칙**과 **RACI 매트릭스 기준**을 정의합니다.

---

## 2. 핵심 원칙: Single Owner

### 2.1 정의

> **모든 문서는 반드시 개인 1명(Owner)에게 귀속된다.**

### 2.2 이유
- 책임 소재 명확화
- 의사결정 지연 방지
- 품질 관리 일원화

### 2.3 금지 사항
- ❌ 팀/부서를 Owner로 지정
- ❌ 복수 인원을 공동 Owner로 지정
- ❌ Owner 미지정 상태로 Approved 전환

---

## 3. Owner 역할과 책임

### 3.1 주요 책임

| 책임 | 설명 |
|------|------|
| **내용 정확성** | 문서 내용이 정확하고 최신 상태 유지 |
| **품질 보증** | Approved 전 품질 기준 충족 확인 |
| **변경 승인** | CHG(변경요청) 검토 및 승인/반려 |
| **폐기 결정** | Deprecated/Archive 전환 결정 |
| **이해관계자 소통** | 관련자에게 변경사항 전파 |

### 3.2 권한

| 권한 | 설명 |
|------|------|
| **최종 결정권** | 문서 내용에 대한 최종 결정 |
| **승인 권한** | Draft → Approved 전환 승인 |
| **대리자 지정** | Deputy 지정 및 해제 |

---

## 4. Deputy (대리자)

### 4.1 정의

> **Owner 부재 시 Owner의 역할을 대행하는 지정된 개인**

### 4.2 지정 조건
- Owner가 명시적으로 지정
- 문서 메타데이터에 기록
- 1명만 지정 가능

### 4.3 Deputy 권한

| 상황 | Deputy 권한 |
|------|------------|
| Owner 휴가/출장 | 전체 권한 대행 |
| Owner 퇴사 전 | 인수인계 수령자 역할 |
| 긴급 상황 | CHG 긴급 승인 가능 |

### 4.4 Deputy가 없는 경우
- **Approved 문서**: 변경 불가, Owner 복귀 대기
- **Draft 문서**: 작업 중단, Owner 복귀 후 재개

---

## 5. 인수인계

### 5.1 인수인계 발생 상황

| 상황 | 처리 방법 |
|------|----------|
| Owner 퇴사 | 필수 인수인계 |
| Owner 부서 이동 | 해당 도메인 문서 인수인계 |
| Owner 역할 변경 | 업무 범위 벗어난 문서 인수인계 |

### 5.2 인수인계 절차

```
1. 현 Owner가 인수자 지정
2. 인수자 승인 확인
3. 문서 메타데이터 Owner 변경
4. 인수인계 기록 (변경 이력에 기재)
5. 관련자에게 Owner 변경 통보
```

### 5.3 인수인계 문서 기록

문서 메타데이터에 기록할 내용:

```markdown
## 변경 이력
| 날짜 | 내용 | 작성자 |
|------|------|--------|
| 2026-01-06 | Owner 변경: 홍길동 → 김철수 | 홍길동 |
```

## 6. RACI 매트릭스

### 6.1 정의

| 역할 | 영문 | 설명 |
|------|------|------|
| **R** | Responsible | 실제 작업 수행자 |
| **A** | Accountable | 최종 책임자 (승인권자) |
| **C** | Consulted | 사전 협의 대상 |
| **I** | Informed | 결과 통보 대상 |

### 6.2 Process Forge에서의 적용

| 활동 | Owner | Deputy | 이해관계자 |
|------|-------|--------|-----------|
| 문서 작성 | R, A | C | I |
| 문서 검토 | A | R | C |
| CHG 승인 | A | R (부재 시) | I |
| 폐기 결정 | A | C | I |

### 6.3 RACI 원칙
- 모든 활동에 **A(Accountable)는 1명만** 존재
- A는 항상 **Owner**
- R은 Owner 또는 Owner가 위임한 자

## 7. 문서 메타데이터 형식

### 7.1 필수 항목

```markdown
---
docId: PRC-planops-001
title: 계획 운영 프로세스
owner: 홍길동
deputy: 김철수
status: Draft
created: 2026-01-06
updated: 2026-01-06
---
```

### 7.2 항목 설명

| 항목 | 필수 | 설명 |
|------|------|------|
| `docId` | O | 문서 식별자 |
| `title` | O | 문서 제목 |
| `owner` | O | Owner 이름 |
| `deputy` | X | Deputy 이름 (지정 시) |
| `status` | O | 문서 상태 |
| `created` | O | 최초 작성일 |
| `updated` | O | 최종 수정일 |

## 9. Owner 미지정 문서 처리

### 9.1 발견 시 조치

```
1. 해당 문서를 Draft 상태로 변경
2. 도메인 관련 책임자에게 Owner 지정 요청
3. 7일 내 Owner 미지정 시 Archive 검토
```

### 9.2 예외
- `.agent/` 내 메타 규칙 파일: 프로젝트 관리자가 Owner
- 템플릿 파일: 프로젝트 관리자가 Owner
