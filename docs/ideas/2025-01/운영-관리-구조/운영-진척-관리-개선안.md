> 작성자: Claude

## 개선 배경

원본 아이디어 문서는 전체적으로 잘 구조화되어 있으나, 프로젝트의 기존 규칙(MASTER.md, document-schema.md, change-control.md)과 정합성을 높이고 실무 적용성을 강화하기 위한 개선이 필요함.

---

## 개선 포인트

### 1. Single Owner 원칙 명확화

**현재 문제**
- 역할 정의(4장)에 PM/PO, Tech Owner 등이 있으나 Phase별 "단일 Owner"가 명시되지 않음
- MASTER.md의 "Single Owner" 원칙과 불일치

**개선안**
- 5.3.C) Phases 속성에 "Owner (단일)" 명시 추가
- 운영 룰 추가: "모든 Phase는 반드시 1명의 Owner를 지정해야 하며, 공동 Owner는 허용하지 않는다"

---

### 2. Rollup 구조 단순화

**현재 문제**
- Cycle Status rollup이 3단계(Task → Phase → Initiative → Cycle)까지 연결
- Notion 성능 저하 및 유지보수 복잡도 증가 우려

**개선안**
- 2단계 rollup으로 단순화: Phase → Cycle (Initiative 생략)
- 또는 Ops Tasks에서 직접 Cycle relation 추가 (역정규화)

```
AS-IS: Ops Tasks → Phase → Initiative → Cycle
TO-BE: Ops Tasks → Phase → Cycle (Initiative는 별도 참조)
```

---

### 3. Re-plan 빈도 명확화

**현재 문제**
- "격주 또는 월 1회"로 모호하게 정의됨
- 팀마다 다르게 해석할 가능성

**개선안**
- 기본값 명시: "월 1회, 매월 첫째 주 금요일"
- 예외 조항: "Cycle Owner 판단 하에 격주로 조정 가능 (단, ADR 기록)"

---

### 4. docId 생성 규칙 연결

**현재 문제**
- docId가 핵심 연결키로 사용되지만, 생성 규칙이 본 문서에 없음
- 신규 사용자가 docId 네이밍을 알 수 없음

**개선안**
- 3.2절에 다음 내용 추가:

```markdown
## 3.4 docId 네이밍 규칙

docId는 `document-schema.md`의 규칙을 따른다.

| 유형 | 형식 | 예시 |
|------|------|------|
| PRD | `PRD-{domain}-{seq}` | PRD-planops-001 |
| QAP | `QAP-{domain}-{seq}` | QAP-planops-001 |

> 상세: `.agent/rules/document-schema.md` 참조
```

---

### 5. 변경통제 중복 제거

**현재 문제**
- 8장 변경통제가 `change-control.md`와 내용 중복
- 두 곳에서 관리 시 불일치 발생 가능

**개선안**
- 8장을 다음과 같이 축소:

```markdown
# 8. 변경통제 (Change Control)

본 프로세스의 변경통제는 `change-control.md`를 따른다.

**핵심 원칙 요약**:
- Approved 문서는 CHG로만 변경
- Phase/Task 일정 재조정은 Re-plan 슬롯에서만 수행

> 상세: `.agent/rules/change-control.md` 참조
```

---

### 6. Gate 조건 체크리스트화

**현재 문제**
- 7.3 Spec Gate 조건이 산문 형태로 되어 있어 검증 어려움

**개선안**
- 체크리스트 형태로 변환:

```markdown
### Spec Gate 체크리스트

- [ ] PRD docId 존재 (SSOT 링크 연결됨)
- [ ] 상태/권한/예외 시나리오 포함
- [ ] 데이터 영향(필드/검증 규칙) 명시
- [ ] QA 핵심 TC 또는 QAP docId 링크 존재
- [ ] Tech Owner 리뷰 완료
```

---

### 7. 용어 정의 섹션 추가

**현재 문제**
- Active Cycle, Gate, Phase 등 용어가 정의 없이 사용됨
- 신규 팀원 온보딩 어려움

**개선안**
- 문서 말미에 용어 정의 섹션 추가:

```markdown
# 부록: 용어 정의

| 용어 | 정의 |
|------|------|
| Cycle | 분기/월/스프린트 등 기간 단위 컨테이너 |
| Initiative | 하나의 큰 계획(에픽) 단위 |
| Phase | 실제 일정이 있는 작업 단계 (Spec/Dev/QA/Release) |
| Gate | 다음 단계 진입 조건 |
| Active Cycle | 현재 운영 중인 기간 (Status = Active) |
| docId | 문서 고유 식별자 |
```
