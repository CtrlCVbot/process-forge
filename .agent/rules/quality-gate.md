## 1. 개요

이 규칙은 문서 상태 전환 시 충족해야 할 **품질 기준**을 정의합니다.

### 1.1 목적
- 문서 품질의 일관성 보장
- 불완전한 문서의 승인 방지
- 검토 기준 표준화

### 1.2 적용 범위
- 모든 문서 유형 (POL, PRC, RUL, CHG)
- 모든 상태 전환 시점

---

## 2. 상태별 게이트

### 2.1 게이트 정의

| 전환 | 게이트명 | 필수 조건 |
|------|----------|----------|
| Draft → Review | Review Gate | 필수 섹션 존재, Owner 지정 |
| Review → Approved | Approval Gate | 모든 필수 항목 충족, 검토 완료 |
| Approved → Deprecated | Deprecation Gate | 대체 문서 명시, 이해관계자 통보 |

### 2.2 게이트 통과 기준

#### Review Gate (Draft → Review)
- [ ] 필수 섹션 모두 존재
- [ ] Owner 지정 (개인 1명)
- [ ] docId 형식 준수
- [ ] 제목과 목적 명확

#### Approval Gate (Review → Approved)
- [ ] Review Gate 조건 충족
- [ ] 모든 필수 항목 작성 완료
- [ ] Owner 검토 완료
- [ ] 품질 점수 7점 이상

#### Deprecation Gate (Approved → Deprecated)
- [ ] 폐기 사유 명시
- [ ] 대체 문서 지정 (있는 경우)
- [ ] 이해관계자 통보 완료
- [ ] 유예 기간 설정 (기본 30일)

## 3. 문서 유형별 체크리스트

### 3.1 POL (정책) 체크리스트

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 목적 | ✅ | 정책 존재 이유 |
| 적용 범위 | ✅ | 누구에게/어디에 적용되는지 |
| 핵심 원칙 | ✅ | 정책의 핵심 원칙 나열 |
| 예외 사항 | ⚠️ | 예외 (없으면 "없음" 명시) |
| 관련 문서 | ⚠️ | 연관된 PRC/RUL 참조 |
| Owner | ✅ | 개인 1명 |
| 변경 이력 | ⚠️ | 주요 변경 기록 |

### 3.2 PRC (프로세스) 체크리스트

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 목적 | ✅ | 프로세스 목표 |
| 적용 범위 | ✅ | 적용 대상/상황 |
| 입력 | ✅ | 프로세스 시작 조건/자료 |
| 출력 | ✅ | 프로세스 산출물 |
| 절차 | ✅ | 단계별 활동 |
| RACI 매트릭스 | ✅ | 역할별 책임 |
| 예외 처리 | ⚠️ | 예외 상황 처리 방법 |
| 에스컬레이션 | ⚠️ | 문제 발생 시 상위 보고 경로 |
| 측정 지표 | ⚠️ | KPI 또는 완료 기준 |
| Owner | ✅ | 개인 1명 |

### 3.3 RUL (규칙) 체크리스트

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 목적 | ✅ | 규칙 존재 이유 |
| 적용 범위 | ✅ | 적용 대상/상황 |
| 규칙 정의 | ✅ | 구체적 규칙 내용 |
| 예시 | ⚠️ | 규칙 적용 예시 |
| 예외 사항 | ⚠️ | 예외 (없으면 "없음" 명시) |
| 위반 시 조치 | ⚠️ | 규칙 위반 시 대응 |
| Owner | ✅ | 개인 1명 |

### 3.4 CHG (변경요청) 체크리스트

| 항목 | 필수 | 설명 |
|------|:----:|------|
| 변경 대상 | ✅ | 원본 문서 docId |
| 변경 사유 | ✅ | 왜 변경이 필요한지 |
| AS-IS | ✅ | 현재 내용 |
| TO-BE | ✅ | 변경될 내용 |
| 영향도 분석 | ✅ | 영향 범위 및 수준 |
| 요청자 | ✅ | 변경 요청자 |
| Owner 승인 | ✅ | 원본 문서 Owner 승인 |

## 4. 품질 점수 기준

### 4.1 점수 산정 방식

```
품질 점수 = (충족 필수 항목 / 전체 필수 항목) × 7 + (충족 권장 항목 / 전체 권장 항목) × 3
```

### 4.2 등급 기준

| 점수 | 등급 | 상태 전환 | 조치 |
|------|:----:|:--------:|------|
| 9-10 | A | ✅ 가능 | 즉시 승인 가능 |
| 7-8 | B | ⚠️ 조건부 | Owner 판단 하에 승인 |
| 5-6 | C | ❌ 불가 | 수정 후 재검증 필요 |
| 1-4 | D | ❌ 불가 | 재작성 권고 |

### 4.3 필수 항목 미충족 시
- 필수 항목 1개라도 미충족 시 **최대 등급 B**
- 필수 항목 2개 이상 미충족 시 **최대 등급 C**

## 5. 게이트 통과 절차

### 5.1 절차 흐름

```
문서 작성 완료
      ↓
자가 점검 (체크리스트)
      ↓
governance-auditor 스킬 검증
      ↓
  ┌───┴───┐
통과      미통과
  ↓         ↓
상태 전환   수정 후 재검증
```

### 5.2 자가 점검

1. 해당 문서 유형의 체크리스트 확인
2. 모든 필수 항목(✅) 충족 여부 점검
3. 권장 항목(⚠️) 가능한 충족
4. 미충족 항목 보완

### 5.3 도구 활용 검증

```
governance-auditor 스킬 호출
→ 자동 체크리스트 검증
→ 품질 점수 산출
→ 개선 권고 제시
```

## 6. 품질 예외

### 6.1 예외 허용 조건

- 긴급 상황 (업무 중단 위험)
- Owner 명시적 승인
- 사후 보완 계획 수립

### 6.2 예외 처리 절차

1. 예외 사유 문서화
2. Owner 승인 획득
3. 상태 전환 진행
4. 30일 내 품질 보완
5. 재검증 실시

### 6.3 예외 제한

- 등급 D (1-4점) 문서는 예외 불가
- 필수 항목 3개 이상 미충족 시 예외 불가
- 동일 문서 2회 이상 예외 적용 금지
