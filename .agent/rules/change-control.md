# change-control.md - 변경 통제 규칙

> **상위 규칙**: MASTER.md
> **버전**: 1.0
> **상태**: Draft

---

## 1. 개요

이 규칙은 Process Forge 프로젝트의 **Approved 문서 변경 절차**와 **CHG(변경요청) 프로세스**를 정의합니다.

---

## 2. 핵심 원칙

### 2.1 Approved 문서 수정 금지

> **Approved 상태의 문서는 직접 수정할 수 없다.**

### 2.2 변경 통제 이유
- 문서 안정성 보장
- 변경 이력 추적
- 영향도 사전 분석
- 이해관계자 사전 통보

### 2.3 적용 범위

| 문서 상태 | 직접 수정 | CHG 필요 |
|----------|----------|---------|
| Draft | O | X |
| Review | O (Owner만) | X |
| **Approved** | **X** | **O** |
| Deprecated | X | X (변경 불가) |

---

## 3. CHG (변경요청) 프로세스

### 3.1 프로세스 흐름

```
변경 필요 발생
      ↓
CHG 문서 작성
      ↓
영향도 분석
      ↓
Owner 검토
      ↓
  ┌───┴───┐
승인      반려
  ↓         ↓
원본 수정   CHG 종료
  ↓
수정 완료
  ↓
이해관계자 통보
```

### 3.2 단계별 상세

| 단계 | 수행자 | 활동 |
|------|--------|------|
| 1. CHG 작성 | 요청자 | CHG 문서 작성, 변경 사유 명시 |
| 2. 영향도 분석 | 요청자 | 변경 영향 범위 파악 |
| 3. Owner 검토 | Owner | CHG 내용 검토, 승인/반려 결정 |
| 4. 원본 수정 | Owner/지정자 | Approved 문서 직접 수정 |
| 5. 통보 | Owner | 이해관계자에게 변경 완료 통보 |

---

## 4. CHG 문서 작성

### 4.1 CHG docId 형식

```
CHG-{원본domain}-{원본seq}-{변경seq}
```

예시:
- `CHG-planops-001-001`: PRC-planops-001의 첫 번째 변경요청
- `CHG-planops-001-002`: PRC-planops-001의 두 번째 변경요청

### 4.2 CHG 문서 필수 항목

```markdown
---
docId: CHG-planops-001-001
title: PRC-planops-001 변경요청 - [변경 요약]
target: PRC-planops-001
requestor: 요청자 이름
owner: 원본 문서 Owner
status: Draft
impact: Medium
created: 2026-01-06
---

## 1. 변경 대상
- 문서: PRC-planops-001
- 현재 버전: 1.0
- 현재 상태: Approved

## 2. 변경 사유
[왜 변경이 필요한지 명확히 기술]

## 3. 변경 내용
### 현재 (AS-IS)
[현재 내용]

### 변경 후 (TO-BE)
[변경될 내용]

## 4. 영향도 분석
- 영향 범위: [영향받는 문서/프로세스 목록]
- 영향 수준: High / Medium / Low

## 5. 승인
- [ ] Owner 승인
- 승인일:
- 승인자:
```

## 5. 영향도 분류

### 5.1 분류 기준

| 수준 | 기준 | 예시 |
|------|------|------|
| **High** | 핵심 프로세스 변경, 다수 문서 영향 | 정책 수정, 승인 절차 변경 |
| **Medium** | 일부 프로세스 영향, 제한적 범위 | 단계 추가/삭제, 담당자 변경 |
| **Low** | 단순 수정, 영향 최소 | 오타 수정, 표현 명확화 |

### 5.2 영향도별 처리

| 수준 | 검토 기간 | 승인 절차 |
|------|----------|----------|
| High | 5 영업일 | Owner + 상위 승인권자 |
| Medium | 3 영업일 | Owner |
| Low | 1 영업일 | Owner |

## 6. 긴급 변경 경로

### 6.1 긴급 변경 조건
- 업무 중단 위험
- 보안/컴플라이언스 이슈
- 법적/규제적 요구사항

### 6.2 긴급 변경 절차

```
1. 긴급 사유 명시한 CHG 작성
2. Owner (또는 Deputy) 즉시 검토
3. 구두 승인 후 문서 수정
4. 24시간 내 CHG 문서 보완
5. 사후 이해관계자 통보
```

### 6.3 긴급 변경 제한
- 긴급 변경은 **High 영향도 변경에만 적용**
- Low/Medium은 일반 절차 따름
- 남용 시 해당 CHG 무효 처리 가능

---

## 7. 승인 및 반려

### 7.1 승인 시

```
1. CHG 문서 상태를 'Approved'로 변경
2. 원본 문서 직접 수정
3. 원본 문서 버전 업데이트
4. 원본 문서 변경 이력에 CHG 참조 추가
5. CHG 문서 상태를 'Completed'로 변경
```

### 7.2 반려 시

```
1. CHG 문서에 반려 사유 기재
2. CHG 문서 상태를 'Rejected'로 변경
3. 요청자에게 반려 통보
```

### 7.3 CHG 상태 정의

| 상태 | 설명 |
|------|------|
| Draft | 작성 중 |
| Review | Owner 검토 중 |
| Approved | 승인됨, 적용 대기 |
| Completed | 원본 문서에 적용 완료 |
| Rejected | 반려됨 |

## 8. 변경 이력 관리

### 8.1 원본 문서 변경 이력 형식

```markdown
## 변경 이력

| 버전 | 날짜 | 변경 내용 | CHG | 작성자 |
|------|------|----------|-----|--------|
| 1.0 | 2026-01-01 | 최초 작성 | - | 홍길동 |
| 1.1 | 2026-01-06 | 3단계 절차 추가 | CHG-planops-001-001 | 홍길동 |
```

### 8.2 버전 규칙
- Major 변경 (High): 1.0 → 2.0
- Minor 변경 (Medium): 1.0 → 1.1
- Patch 변경 (Low): 1.0 → 1.0.1

---

## 9. Draft 문서 직접 수정

### 9.1 Draft 수정 권한

| 수정자 | 권한 |
|--------|------|
| Owner | 자유롭게 수정 가능 |
| Deputy | Owner 위임 시 수정 가능 |
| 기타 | Owner 승인 후 수정 가능 |

### 9.2 Draft 수정 시 주의
- 변경 이력에 주요 변경 기록
- Approved 전환 전 최종 검토

---

## 10. 예외 사항

### 10.1 CHG 없이 수정 가능한 경우
- 명백한 오타 (내용 변경 없음)
- 마크다운 포맷팅 수정
- 링크 URL 업데이트 (내용 동일)

### 10.2 예외 적용 조건
- Owner만 수행 가능
- 변경 이력에 "Minor fix" 기록
- 버전 변경 없음
